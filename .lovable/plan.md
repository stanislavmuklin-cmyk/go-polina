

# Plan: Turning the Wellness App into a Production-Ready AI Platform

## Overview

Transform the current demo app into a fully functional AI-powered wellness platform by connecting Lovable Cloud, enabling Lovable AI, and reworking key modules so that all content (workouts, nutrition, supplements, SOS) is generated by AI based on the user's complete profile.

---

## Phase 1: Enable Backend Infrastructure

### 1.1 Enable Lovable Cloud
- Activate Lovable Cloud for the project (database, edge functions, secrets).

### 1.2 Enable Lovable AI
- Activate the Lovable AI gateway so edge functions can call `google/gemini-3-flash-preview`.

### 1.3 Create Core Edge Function: `ai-generate`
- A single edge function that accepts a `type` parameter (`workouts`, `meals`, `supplements`, `shopping`, `sos`) and the full user profile as context.
- System prompt instructs the AI to act as an evidence-based wellness consultant (RCT, physiology, clinical nutrition) and generate structured JSON responses.
- The user's profile (goal, weight, height, age, gender, fitness level, diet type, complaints, cycle tracking) is always included in the prompt context.
- Uses tool calling to extract structured JSON output (not raw text parsing).
- Handles 429/402 errors and surfaces them to the client.

## Phase 2: Onboarding Improvements

### 2.1 Add Diet Type selection (new step or extend step 3)
- Add a new field `dietType` to `UserProfile`: `"no-restriction" | "keto" | "paleo" | "mediterranean" | "low-carb"`.
- Options: "Без ограничений", "Кето", "Палео", "Средиземноморская", "Низкоуглеводная".
- This is separate from the existing `dietPreferences` (allergies/restrictions) -- it's the overall dietary approach.

### 2.2 Add Workout Location selection
- Add `workoutLocation` field to `UserProfile`: `"gym" | "home"`.
- Options: "Зал" and "Дома" -- shown on step 3 alongside fitness level.

### 2.3 Sharpen Goal step
- Keep existing 5 goals but add short descriptions so the user clearly understands each one.

## Phase 3: AI-Powered Workouts

### 3.1 Rework `Workouts.tsx`
- On first load (or when no saved plan exists), call the `ai-generate` edge function with `type: "workouts"` and the user profile.
- AI generates a weekly plan (5-7 days) with exercises, sets, reps -- tailored to goal, fitness level, and workout location (gym vs home).
- Display a loading skeleton while generating.
- Store the generated plan in localStorage (keyed by a hash of relevant profile fields).

### 3.2 Regenerate button
- Add a "Regenerate" button at the top of the workouts page.
- Clicking it calls the AI again and replaces the current plan.
- Add a toggle to switch between "Gym" and "Home" -- switching triggers regeneration.

---

## Phase 4: AI-Powered Nutrition (7-day plan)

### 4.1 Rework `Nutrition.tsx` -- Meals tab
- On first load, call `ai-generate` with `type: "meals"` to generate a 7-day meal plan.
- Each meal includes: name, ingredients, calories, protein, fat, carbs, and preparation time.
- Display day selector (Mon-Sun) at the top.
- Each meal card shows: time, meal name, ingredients list, and macros breakdown (kcal / P / F / C).

### 4.2 Per-meal regeneration
- Each meal card gets a "Regenerate" button (refresh icon).
- Clicking it calls the AI to regenerate just that one meal, keeping the rest of the day intact.
- The AI receives the full day context so the replacement fits the daily macro target.

### 4.3 Supplements tab -- AI-generated
- Call `ai-generate` with `type: "supplements"` to generate supplement recommendations based on goal, complaints, and diet.
- Each supplement shows: name, dose, timing, duration, and why it's recommended.
- Per-supplement regenerate button.

### 4.4 Shopping list -- AI-generated
- Call `ai-generate` with `type: "shopping"` based on the current 7-day meal plan.
- Grouped by category (proteins, vegetables, grains, etc.).

---

## Phase 5: AI-Powered SOS

### 5.1 Rework `SOS.tsx`
- Keep the existing SOS cards (titles + emojis) but remove the hardcoded protocol text.
- Cards are now clickable buttons showing only the title.
- When clicked, a modal/drawer opens and calls `ai-generate` with `type: "sos"` and the specific SOS topic.
- AI generates a personalized protocol based on the user's full profile (diet, goal, complaints, etc.).
- Show a loading state while generating, then display the AI response.

## Phase 6: AI Chat Context Upgrade

### 6.1 Update `AskAI.tsx`
- Replace the simulated `setTimeout` responses with real streaming calls to a `chat` edge function.
- The chat edge function includes the full user profile in the system prompt, so the AI always knows the user's parameters.
- Token-by-token streaming for smooth UX.

## Technical Details

### Edge Function: `supabase/functions/ai-generate/index.ts`
- Accepts POST with `{ type, profile, extra }`.
- Builds a system prompt with the user's full profile data.
- Uses tool calling to get structured JSON output (different schemas per type).
- Returns JSON response.

### Edge Function: `supabase/functions/chat/index.ts`
- Accepts POST with `{ messages, profile }`.
- Injects profile into system prompt.
- Streams SSE response back to client.

### Frontend helpers: `src/lib/ai.ts`
- `generateContent(type, profile, extra)` -- calls `ai-generate` function via `supabase.functions.invoke()`.
- `streamChat(messages, profile, onDelta, onDone)` -- handles SSE streaming for the chat.

### Config: `supabase/config.toml`
- Add both functions with `verify_jwt = false`.
